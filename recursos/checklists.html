Checklists RUN y capacitación

Listas rápidas para validar configuración, diseño y soporte.

Checklist maestro de capacitación

Usar este material como hoja de apoyo para sesiones prácticas.

1. Checklist DFS

  • Digital service identificado.
  • Operation code/id identificado.
  • Business channel asociado.
  • Agreement/servicio/cuenta validado.
  • Cargos revisados o simulados.
  • Límites/acumuladores asignados.
  • Flow / business logic conectado.
  • Posting definido si aplica.
  • Logs y trazas disponibles.

2. Checklist límites/acumuladores

  • Servicio digital y operación.
  • Canal de negocio.
  • Monto, moneda y fecha/hora.
  • Acumulador asociado.
  • Límite asociado.
  • Periodo de acumulación.
  • Valor consumido actual.
  • Resultado esperado: permitir, rechazar, SFA/MFA.
  • Respuesta y código de error, si aplica.

3. Checklist Designer

  • Nombre claro del flow.
  • Entrada/salida documentada.
  • Nodos con responsabilidad única.
  • Ramas de error técnico y funcional.
  • Mapeo request/response.
  • Logs de inicio, decisión, llamada externa, error y fin.
  • Correlación propagada.
  • Sin secretos expuestos.
  • Plan de pruebas y rollback/compensación.

4. Checklist integración externa

  • Contrato oficial validado.
  • Autenticación definida de forma segura.
  • Timeout/retry/fallback definidos.
  • Mapeo de errores externos.
  • Idempotencia o consulta de estado para operaciones monetarias.
  • Monitoreo de latencia y códigos externos.
  • Runbook de soporte.

5. Checklist para leer publishers

  • Identificar Project, versión y fecha.
  • Contar flows, servicios digitales, REST operations, posting, disaggregation, acumuladores y parámetros.
  • Detectar secretos o parámetros sensibles y no copiarlos.
  • Para cada flow: entradas, salidas, nodos, EXECAPI, EXECFLOW, condiciones y errores.
  • Para cada DFS: template, estado, operaciones, posting, business channels y acumuladores.
  • Para cada REST operation: método, resource, target flow, content-type, auth y parámetros.
  • Para posting: ordinal, signo, operation, grupo y error step.
  • Registrar pendientes de validar cuando el publisher no indique una regla completa.
Manual interno de capacitación DeUna. No reemplaza documentación oficial ni publishers. No incluye secretos, tokens ni credenciales.
